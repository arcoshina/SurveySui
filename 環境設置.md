# SurveySui — WSL 開發環境設定

照順序執行。每一段都在 **WSL Ubuntu** 終端機裡跑（不是 PowerShell）。

> ⚠️ **重要**：repo 要放在 WSL filesystem (`~/projects/`)，不要從 `/mnt/d/...` 操作 —
> WSL 跨檔案系統的 I/O 會慢 5–10 倍，pnpm install 跟 Playwright 會痛苦到不行。

---

## Step 0 — Ubuntu 基礎環境

```bash
# 更新系統
sudo apt update && sudo apt upgrade -y

# 安裝基本工具
sudo apt install -y build-essential curl git unzip pkg-config libssl-dev ca-certificates

# 設定 git（換成你自己的）
git config --global user.name "wesleyshun2"
git config --global user.email "wesleyshun2@gmail.com"
git config --global init.defaultBranch main
git config --global core.autocrlf input   # 避免 Windows CRLF 汙染
```

## Step 1 — 把 repo 搬到 WSL filesystem

```bash
mkdir -p ~/projects
cd ~/projects

# 從 Windows 磁碟複製過來（保留 .git 歷史）
cp -r /mnt/d/Users/Arco_asus/Documents/GitHub/SurveySui ./SurveySui
cd ~/projects/SurveySui

# 驗證 git 還能用
git status
git log --oneline -5
```

之後所有開發都在 `~/projects/SurveySui` 進行，Windows 那份當備份。

## Step 2 — Node.js 20 + pnpm（用 fnm 管版本）

```bash
# 裝 fnm（比 nvm 快很多）
curl -fsSL https://fnm.vercel.app/install | bash

# reload shell
source ~/.bashrc

# 裝 Node 20 LTS
fnm install 20
fnm default 20
fnm use 20

node -v   # 應該顯示 v20.x.x

# 裝 pnpm
npm install -g pnpm@9
pnpm -v   # 應該顯示 9.x.x
```

## Step 3 — Sui CLI（testnet 版本）

最快的方式是抓官方 binary：

```bash
# 抓 testnet 版 binary
cd ~
SUI_VERSION="testnet-v1.39.3"   # 開發時請去 https://github.com/MystenLabs/sui/releases 看最新 testnet tag
curl -L -o sui.tgz "https://github.com/MystenLabs/sui/releases/download/${SUI_VERSION}/sui-${SUI_VERSION}-ubuntu-x86_64.tgz"

# 解壓並放到 PATH
mkdir -p ~/.local/bin
tar -xzf sui.tgz -C ~/.local/bin sui sui-faucet sui-test-validator 2>/dev/null || tar -xzf sui.tgz -C ~/.local/bin
rm sui.tgz

# 確認 PATH 包含 ~/.local/bin
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# 驗證
sui --version
```

> 如果 release 頁面只看到 `sui-mainnet-*` 跟 `sui-testnet-*`，挑 `testnet` 那個。
> 若 binary 安裝失敗，fallback：`cargo install --git https://github.com/MystenLabs/sui.git --branch testnet sui`（需要 Rust，會編譯 20-40 分鐘）。

### 設定 testnet 環境 + admin 錢包

```bash
# 第一次跑會問你要連哪個網路，選 testnet，URL 填 https://fullnode.testnet.sui.io:443
sui client

# 或手動加：
sui client new-env --alias testnet --rpc https://fullnode.testnet.sui.io:443
sui client switch --env testnet

# 產一個 admin address（記下 mnemonic！）
sui client new-address ed25519 admin

# 查看 address
sui client active-address

# 領 testnet SUI（會發到 active address）
sui client faucet
sui client balance
```

## Step 4 — PostgreSQL（選一種）

### 選項 A：原生安裝（推薦給單人開發）

```bash
sudo apt install -y postgresql postgresql-contrib
sudo service postgresql start

# 建 dev user + database
sudo -u postgres psql <<EOF
CREATE USER surveysui WITH PASSWORD 'dev_password';
CREATE DATABASE surveysui_dev OWER surveysui;
GRANT ALL PRIVILEGES ON DATABASE surveysui_dev TO surveysui;
EOF

# 驗證
psql -h localhost -U surveysui -d surveysui_dev -c "SELECT version();"
```

> WSL 不會自動啟動 postgres。每次開機後跑 `sudo service postgresql start`，
> 或加到 `~/.bashrc`：`sudo service postgresql start 2>/dev/null`

### 選項 B：Docker（推薦給「跟 CI 一致」）

```bash
# 如果還沒裝 Docker Desktop for Windows，先去裝，並在設定打開 WSL2 integration
docker --version   # 確認 wsl 內看得到 docker

# 起一個 postgres 容器
docker run -d --name surveysui-pg \
  -e POSTGRES_USER=surveysui \
  -e POSTGRES_PASSWORD=dev_password \
  -e POSTGRES_DB=surveysui_dev \
  -p 5432:5432 \
  postgres:16
```

**`DATABASE_URL` 兩種都長這樣**：
```
postgresql://surveysui:dev_password@localhost:5432/surveysui_dev
```

## Step 5 — Google OAuth Client（zkLogin 用）

不需要在終端機跑，但現在先去拿好，後面 T2.2 會用到：

1. 開 https://console.cloud.google.com/
2. 建立一個 Project（名字隨意，例如 `SurveySui-dev`）
3. APIs & Services → Credentials → Create OAuth client ID
4. Application type: **Web application**
5. Authorized redirect URIs 先填：
   - `http://localhost:3000/auth/google/callback`
   - `http://localhost:4000/auth/google/callback`
6. 把 `Client ID` 和 `Client Secret` 存好，等下放進 `.env`

## Step 6 — VS Code Remote-WSL

在 Windows 端的 VS Code：

1. 裝 extension：**WSL**（Microsoft 官方）
2. 在 WSL 終端機裡跑：
   ```bash
   cd ~/projects/SurveySui
   code .
   ```
3. VS Code 會自動以 Remote-WSL 模式打開，從此編輯、終端機都在 Linux 環境

建議再裝這幾個 extension（會自動裝在 WSL 端）：
- ESLint
- Prettier
- Move (by Mysten Labs) — Move syntax highlight
- Prisma — schema 高亮 + 自動補全
- Tailwind CSS IntelliSense
