## MVP 方向：區塊鏈上的問卷平台

在 SUI 上利用區塊鏈的特性打造流暢的**問卷激勵平台**，協助初創或 NPO 蒐集輿情

## 一句話定位 & 使用者

- 產品定位：  
  「給小型創業者用的鏈上問卷獎勵系統：用明確、可控且即時的激勵系統，有效率地蒐集回饋。」

- 主要使用者：
  - 小型創業者／電商／自有品牌主：原本就需要問卷＋獎勵來驗證市場。
  - 行銷／顧問接案者：幫客戶跑小型調查，希望獎勵發放更透明可靠。
  - 學術研究者：蒐集學術論文所需的問卷資料。

## MVP 要證明什麼

用最小功能證明三件事：

1. **零門檻產品層**：發起者透過前端完成「設定問卷＋分享 → 看結果」。最重要的是**讓受訪者在不需要入金、錢包餘額為 0 的前提下，透過 Sponsored Transactions (代付 Gas) 順暢完成填答並領取獎勵**，同時透過合約模擬與驗證機制防止惡意行為消耗 Gas 導致專案破產。
2. **去中心化與防女巫**：移除對中心化後端的依賴，問卷資格與防女巫（如 Email 憑證或其他 DID）**全權交由智能合約驗證**。MVP 階段可先發行 `SurveyPass` 物件作為臨時通行證，合約在填答時只驗證而不消耗該物件。
3. **金流層 (代幣經濟)**：填答的獎勵是項目代幣 `SurveySuiReward` 的質押憑證 (`stakedSurveySuiReward`)，受訪者可以使用憑證從 AMM 池領出獎勵代幣。發起者付出 Sui，依 AMM 價格從質押池 mint 出「增發的」獎勵代幣，池裡 Sui 變多帶動代幣價值上漲。當發起者以獎勵代幣產生問卷時，新 mint 的代幣會回到池中，而填答者獲得質押憑證。池子裡的 Sui 只能由管理者領出，手續費也回到專案方手中（可賣出或燒毀）。
4. 問卷**使用 Markdown + metadata 設定**，轉換為智能合約可用的格式，把精力放在「資金流＋獎勵流」做得絲滑無縫。
5. 匿名化投票系統

## MVP 規格總覽

### 1. 角色與錢的流向

- 發起者 (Creator): 創業者／顧問
- 受訪者 (Respondent): 填問卷拿獎勵的人
- 管理者 (Admin): 專案方（收取手續費、管理質押池的 Sui）
- 合約資產：
  - SurveyVault：每個問卷對應一個 on-chain vault object，紀錄問卷狀態與預算。(原 RewardVault)
  - SurveySuiReward：本項目的獎勵代幣。
  - stakedSurveySuiReward：受訪者取得的獎勵（項目代幣的質押憑證）。受訪者可憑此向池子領出 SurveySuiReward。
  - SurveyPass：防女巫用的通行證物件 (如 Email 憑證或 DID)，問卷合約會驗證持有者資格。
  - AMM Pool (質押池)：發起者投入 Sui，Mint 增發的 SurveySuiReward。池中的 Sui 僅供管理者領出。

### 2. 使用者流程（發起者）

**Flow A：建立「問卷＋獎勵」活動**

1. 註冊／登入前台（連錢包）。
2. 新建「問卷活動」：
   - 輸入標題、描述（例如「 Overflow 滿意度調查」）。
   - 編輯問卷 Markdown 檔（MVP 只內建 1個模板 MD 檔）。
     a. MVP: 使用者可文字編輯 Markdown
     b. 基礎: 使用者可編輯 & 預覽
3. 設定獎勵規則與參與條件：
   - 填完一份的獎勵金額與最多發放名額。
   - 設定統計結果如何交付。
   - 設定參與條件：如必須持有 `SurveyPass` 或特定的 Email 憑證/DID 才能填答，交由合約驗證。
4. 連接 Sui 錢包並「注資」：
   - 前端顯示「預估總 Token 消耗」+「平台手續費」（透過 `amm_pool` 價格計算與手續費比例預估）。
   - 發起一筆原子 PTB (Programmable Transaction Block)：
     - **預算兌換**：呼叫 `amm_pool::invest_and_mint` 將發起者投入的 Sui 注資到 AMM 池中，池中 Sui 增加並鑄造 `SurveySuiReward` (SSR) 作為儲備留在池內，同時 1:1 鑄造等額的 `stackedSurveySuiReward` (sSSR) 代幣傳回給發起者。- **建立金庫**：呼叫 `survey_vault::create` 將上述 sSSR 代幣作為問卷預算存入新建的 `SurveyVault` 金庫物件中，此時合約會自動扣除 0.3% 手續費至管理者的 `admin_treasury`；金庫內會記錄單筆獎勵、上限名額與防女巫條件。- **註冊問卷**：呼叫 `survey_registry::register` 將金庫與問卷的加密內容 Hash 進行鏈上綁定與註冊，並發送 `SurveyRegistered` 事件。
       結果：產生一個「活動連結」，前端 URL 對應鏈上的 `SurveyVault` ID，且問卷內容與預算已完全安全上鏈。

### 3. 使用者流程（受訪者）

**Flow B：填答 & 拿獎勵**

1. 點擊發起者分享的問卷連結，連到前端網站（RWD）。
2. 引導社群登入/連錢包。受訪者**不需要入金，錢包餘額可為 0**。
   - 若尚未持有 `SurveyPass`，前端會發起 PTB 請求代付 Gas 來 Mint 一張通行證 (驗證 Email/DID)。
3. 進入問卷並填答：
   - 前端檢查鏈上的 `SurveyVault` 是否尚有剩餘名額，以及地址是否已填答過。
   - 填寫完畢，前端構建 PTB 並將問卷內容打包。
4. 受訪者送出問卷 (透過 Sponsored Transactions 代付 Gas)：
   - 前端將 PTB 送至 Gas Station (贊助節點) 請求代付簽名。
   - **防惡意消耗機制**：Gas Station 會先在鏈上模擬 (Dry Run)。如果受訪者資格不符 (無效的 `SurveyPass` 或重複填答)，合約會報錯，Gas Station 拒絕簽名，避免浪費專案方的 Gas。
   - 模擬成功後，取得 Gas 簽名並廣播上鏈。
   - **智能合約驗證**：合約純鏈上驗證 `SurveyPass` (只驗證，不消耗)，將問卷結果加密上鏈，並自動從 `SurveyVault` 紀錄發放狀態。
   - 合約自動派發 `stakedSurveySuiReward` (質押憑證) 給受訪者的地址。
5. 受訪者在鏈上資訊確認：
   - 收到 `stakedSurveySuiReward`，可隨時拿回 AMM Pool 兌換出 `SurveySuiReward`。

### 4. 使用者流程（活動結束後）

**Flow C：發起者收尾 & 資金管理**

1. 活動進行中，在 dashboard 裡看到：
   - 回覆數、完成率、基本統計表（單選長條圖、量表平均分數等）。
   - 各種設定：已發放份數、已花費獎勵金額、剩餘 `SurveyVault` 額度、結束時間。
2. 達成結束條件：
   - 時間超過設定 或 `SurveyVault` 預算用盡。
3. 一鍵「結束活動」按鈕：
   - 發起 PTB：
     - 關閉 `SurveyVault`，不再接受新回覆。
     - 若有剩餘預算，則依據機制退還剩餘價值給發起者。
4. 送出結果：
   - 依照設定方法送出結果 (鏈上解密讀取或匯出報告等)。
