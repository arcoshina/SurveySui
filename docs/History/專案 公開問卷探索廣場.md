# 公開問卷探索廣場 + 依語言篩選

## Context（為什麼做這個）

目前所有問卷在鏈上都是 **shared object**，任何人技術上都查得到其存在，但**標題、說明、題目全被加密**在 `encrypted_content` 裡，金鑰只放在分享連結的 URL fragment。因此問卷實質上是「連結限定/私密」模型 —— 沒有連結就無從得知問卷內容。

本專案欲新增一個**公開探索廣場**：讓「訪客」（未連錢包）或一般使用者能瀏覽公開問卷，並**依訪客介面語言進行篩選**，避免隨機盲答。

核心洞察：**要放到公開廣場的問卷，本來就該被所有人看到，加密它沒有意義。**
因此「公開到廣場」= 「**內容不加密**」。內容是明文時，前端與任何人均可直接讀取標題、題目並自動偵測或讀取設定的語言。

基於此洞察：
- **不需修改 Move 合約**（合約把內容當不透明 `vector<u8>`，明文/密文都收）
- **不需新增 title/language 鏈上欄位**（直接從明文 markdown 取得）
- **不需 BFF 新設複雜的快取端點**（改由前端直接向 Sui RPC 進行高效分頁查詢與篩選）
- **忽略 Walrus 問卷**（為了保證廣場載入的高效能與子秒級回應，只展示小於 10KB 且內容直接儲存在鏈上 `encrypted_content` 的明文問卷）

## 設計決策

| 決策項目 | 實作方案 |
|---|---|
| **架構模型** | **前端自主查詢模式**（前端直接向 Sui RPC 查詢事件與物件，不依賴 BFF 提供廣場 API，減輕伺服器負擔） |
| **明文儲存** | **不加密選項**，內容明文存鏈上（沿用 `encrypted_content` 欄位裝明文 bytes） |
| **上架模式** | **在「注資發布頁」取消勾選「加密問卷內容」代表公開**（問卷建立頁 `CreatePage.tsx` 不進行任何變更，僅在 `FundPage.tsx` 加強提示與安全性警告） |

## 問卷二分法與儲存格式

### 1. 內容 Blob 格式標記（無合約變更）
在 `encrypted_content` 內容 blob 最前面加 **1 byte 版本標記**：
- `0x00` → 後面是**明文 markdown**（公開問卷）
- `0x01` → 後面是**現有加密格式** `[32B x25519_pub | 12B iv | ciphertext]`（私密問卷）

### 2. 比較表

| | 私密問卷 (現狀不變) | 公開問卷 (新規格) |
|---|---|---|
| **內容 blob** | `[0x01 \| 加密 payload]` | `[0x00 \| 明文 markdown]` |
| **誰能看** | 只有拿到 URL 金鑰的人 | 任何人 |
| **URL fragment 金鑰** | 有 (Hash 中包含 AES 金鑰) | 無 (不需要金鑰即可開啟) |
| **上廣場條件** | 否 | 是 (需小於 10KB 且未被 Walrus 託管) |

> 註：**填答限制**仍受 `min_tier` / SurveyPass / NFT 管制不變 —— 公開只代表「內容可瀏覽」，答題與領獎仍需持有對應憑證。

## 查詢與篩選邏輯 (Frontend-Only Workflow)

### 1. Sui RPC 掃描流程
當使用者進入 `/explore` 頁面時，前端執行以下流程：
1. **降序查詢註冊事件**：呼叫 `suiClient.queryEvents` 查詢 `${PACKAGE_ID}::survey_registry::SurveyRegistered`，設定 `order: 'descending'`（由新到舊）。
2. **時間窗口截斷（7 天）**：遍歷事件，檢查 `registered_at_ms`。一旦發現事件發布時間早於 7 天前（`Date.now() - 7 * 86400000`），**立即停止分頁查詢**。這確保了只拉取最近一週的活動，且 RPC 呼叫次數極少。
3. **批次加載物件**：將篩選出 7 天內發布的 `survey_id` 與 `vault_id` 分別打包，使用 `multiGetObjects`（以 50 個為一組進行 Chunking）批次向 RPC 查詢 Survey 物件與 Vault 物件。
4. **過濾與解析**：
   - **狀態過濾**：若 Vault 物件 `status === 1` (已關閉)、`claimed_count >= max_responses` (已額滿)、或當前時間已超過 `deadline_ms` (已過期)，則不予展示。
   - **儲存過濾**：若 `survey_blob_id` 存在（被 Walrus 託管），為保證效能，直接跳過。
   - **格式校驗**：讀取 `encrypted_content` 的首 Byte，若不為 `0x00` 或後續內容無法解析出有效的 YAML frontmatter markdown，直接忽略（拋棄舊測試資料與防止碰撞）。

### 2. 填答狀態過濾
- 若使用者已連接錢包，前端會檢查使用者是否已填過該問卷。
- 檢查方式：透過查詢該 Vault 在鏈上的 `SurveyClaimed` 填答事件，判斷該錢包地址是否已在 claimed 名單中，若是則在廣場中隱藏或標示為「已填寫」。

## 語言與 UI 空狀態規則

1. **問卷語系來源優先序**：
   - 優先讀取 frontmatter 中的 `language`（一律採用 ISO 639-1 小寫代碼）。
   - 若未填寫，則呼叫 `detectLang` 以 Unicode Script 比例自動偵測，並轉為小寫 ISO 639-1 代碼。
2. **UI 篩選與空狀態提示**：
   - 當 UI 介面為日文（`ja`）時，廣場預設過濾並顯示日文問卷。若沒有合格日文問卷，顯示灰色字體的**無日文問卷提示**（日文文案）。
   - 使用者可以手動切換至英文（`en`）介面，此時顯示英文問卷。若英文也沒有問卷，則顯示灰色字體的**無問卷提示**（英文文案）。
   - 提供「顯示所有語言（Show All Languages）」切換開關，允許訪客無視語系過濾瀏覽所有公開問卷。

## UI/UX 設計與視覺規劃（嚴格遵循 Style Guide）

本專案之所有 UI 設計細節必須完全對齊 [Style_Guide.md](file:///d:/Users/Arco_asus/Documents/GitHub/SurveySui/docs/Style_Guide.md) 的規範：

### 1. 探索廣場頁面 (`ExplorePage.tsx`)
- **字型與排版**:
  - 全域使用 `Noto Sans TC`。
  - 頁面標題字型大小與粗細設定為 `.text-h1` (`text-2xl font-normal text-slate-900 dark:text-neutral-100`)。**不使用任何文字漸層與 bold/semibold 字重**。
  - 輔助說明使用 `.text-muted`。
- **篩選工具列**:
  - **語系選擇器**：使用標準圓角膠囊按鈕 (`px-3 py-1 text-sm font-normal rounded-xl`)，**無按鈕陰影，無背景漸層**。
    - 亮色選中：`bg-blue-50 text-blue-700`；暗色選中：`bg-blue-900/30 text-blue-400`。
    - 未選中：`text-neutral-600 dark:text-neutral-300 hover:bg-neutral-100 dark:hover:bg-neutral-800`。
  - **「顯示全部語系」開關**：精簡的開關元件（無陰影），與工具列平齊。
- **加載狀態**:
  - 骨架屏卡片使用標準容器背景與淡灰閃爍，不使用複雜的外部漸層。
- **問卷卡片網格與卡片設計**:
  - 響應式網格（`grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6`）。
  - **卡片容器**：採用 Style Guide 規範之標準容器樣式，**無玻璃擬態（不使用 backdrop-blur 等）、無卡片 Hover 上浮動畫、無陰影**：
    - 亮色模式：`bg-slate-100 border border-slate-300 rounded-2xl p-5`
    - 暗色模式：`bg-neutral-950 border border-neutral-800 rounded-2xl p-5`
  - **卡片內文字**：
    - 標題字型大小為 `.text-h3` (`text-lg font-normal`)。**禁止使用粗體**。
    - 說明內文為 `.text-body` (`text-base font-normal`)。
    - 語言標籤與其他指標使用 `.text-muted`。
  - **徽章與晶片 (Badges & Chips)**：
    - 格式統一為 `px-2 py-0.5 rounded-full text-sm font-normal border`。
    - **語言/選填徽章**：`.chip-optional`（亮色：`bg-slate-100 border-slate-200 text-slate-500`；暗色：`dark:bg-neutral-800 dark:border-neutral-700 dark:text-neutral-400`）。
    - **公開徽章 (Tier 0)**：使用綠色 `.badge-direct`（亮色：`bg-emerald-50 border-emerald-200 text-emerald-800`；暗色：`dark:bg-rose-600/20...` 對齊規範中綠色配色）。
    - **獎勵標記**：中性字型 `.text-body` 與中性底色，標示「X SSR」。
- **無問卷提示 (Empty State)**:
  - 提示字級為 `.text-muted`（灰字，最小 `text-sm`，無 Emoji）。

### 2. 注資發布頁面 (`FundPage.tsx`)
- **發布警告提示框 (Warning Alert)**:
  - 當「加密問卷內容」複選框（`encrypt-survey`）為**未勾選**狀態時，下方動態展示警告框。
  - 警告框必須為標準的 `.alert-error`（無 Emoji，改用 Lucide 的 `AlertTriangle`，大小為 `14` 或 `16`）：
    - 亮色模式：`bg-rose-50 text-rose-800 border-rose-100 p-4 rounded-xl text-sm font-semibold border flex items-center gap-2`。
    - 暗色模式：`bg-rose-900/20 text-rose-300 border-rose-900/30 p-4 rounded-xl text-sm font-semibold border flex items-center gap-2`。
