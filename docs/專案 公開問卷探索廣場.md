# 公開問卷探索廣場 + 依語言篩選

## Context（為什麼做這個）

目前所有問卷在鏈上都是 **shared object**，任何人(含 BFF)技術上都查得到其存在，
但**標題、說明、題目全被加密**在 `encrypted_content` 裡，金鑰只放在分享連結的 URL
fragment。因此問卷實質上是「連結限定/私密」模型 —— 沒有連結就無從得知問卷內容。

使用者想新增一個**公開探索廣場**：讓「訪客」(未連錢包)能瀏覽公開問卷，並**依訪客系統
語言自動篩選**，避免隨機盲答。

核心洞察(經使用者確認)：**要放到公開廣場的問卷，本來就該被所有人看到，加密它沒有意義。**
因此「公開到廣場」= 「**內容不加密**」。內容是明文時，BFF 可直接讀標題、題目並自動偵測語言，
於是：

- **不需修改 Move 合約**(合約把內容當不透明 `vector<u8>`，明文/密文都收)
- **不需新增 title/language 鏈上欄位**(BFF 直接讀明文 markdown)
- **語言自動偵測可行**(只有明文才能偵測 —— 對上使用者選的「自動偵測內容語言」)

## 設計決策(已與使用者確認)

| 決策 | 選擇 |
|---|---|
| 明文 metadata 放哪 | **公開問卷不加密**，內容明文存鏈上(沿用 `encrypted_content` 欄位裝明文 bytes) |
| 上架模式 | **創建者勾選「公開到廣場」才上架**(opt-in，預設關=維持現有加密私密問卷) |
| 語言來源 | **BFF 自動偵測內容語言**，存成 **locale code**(非 ZH/EN 二分，見下方多語系設計) |
| 訪客系統語言 | 前端依 `navigator.language` 初始化介面語言，廣場預設以此語言篩選、可切「全部語言」 |

> **未來會新增其他語系支援** —— 因此語言相關設計一律以 **locale code**(如 `zh`/`en`/`ja`/`ko`)
> 為單位，而非寫死 ZH/EN 二分，讓加語系時只需擴充偵測規則與篩選清單、不必重構。詳見「多語系擴充設計」。

問卷的二分法：

| | 私密問卷(現狀不變) | 公開問卷(新增) |
|---|---|---|
| 內容 blob | `[0x01 | 加密 payload]` | `[0x00 | 明文 markdown]` |
| 誰能看 | 只有拿到 URL 金鑰的人 | 任何人(含 BFF) |
| URL fragment 金鑰 | 有 | 無(不需要) |
| 上廣場 | 否 | 是 |

> 註：**填答**仍受 `min_tier` / SurveyPass 管制不變 —— 公開只是「可瀏覽」，答題與領獎仍需對應
> 等級憑證。廣場卡片上顯示 tier 徽章,讓訪客先知道門檻。

## 內容 blob 格式標記(無合約變更)

在內容 blob 最前面加 **1 byte 版本標記**：
- `0x00` → 後面是**明文 markdown**(公開問卷)
- `0x01` → 後面是**現有加密格式** `[32B x25519_pub | 12B iv | ciphertext]`(私密問卷)

合約 `register()` 不看這個 byte(對它就是 bytes)，純屬前端/BFF 約定。
舊測試問卷(無標記)由 BFF / SurveyPage 當作加密格式 fallback 處理(testnet 開發階段可接受)。

### 3. BFF — 廣場端點

- **新模組 `bff/src/explore/handler.ts`**：`registerExploreRoutes(app, deps)`，於
  [bff/src/app.ts](bff/src/app.ts) 註冊(比照 `registerStatsRoutes` 模式)。
  - `GET /api/surveys/explore?lang=ZH&cursor=...`
  - 流程(整份目錄快取 ~60s，非每訪客重算)：
    1. 分頁 `queryEvents` `SurveyRegistered` → 收集 `survey_id / vault_id / min_tier /
       question_count / registered_at_ms`。可抽共用 `fetchRegisteredSurveys()`,
       消除 [bff/src/purge/purgeTask.ts](bff/src/purge/purgeTask.ts) 與前端的重複查詢。
    2. `multiGetObjects(survey_ids)` 批次讀 Survey 物件：跳過 `status==ARCHIVED`；
       讀內容 blob 第一個 byte，**只留 `0x00` 公開問卷**；解析 markdown frontmatter 取標題、
       自動偵測語言(`detectLang(md)` 回傳 **locale code**，見「多語系擴充設計」)。
    3. `multiGetObjects(vault_ids)` 取截止時間/獎勵/開放狀態，濾掉已關閉/過期者。
  - 回傳 `{ surveys: [{ surveyId, vaultId, title, language, minTier, questionCount,
    deadlineMs, perResponse }], nextCursor }`；有 `?lang` 則 server 端先濾語言。
- **新工具 `bff/src/explore/detectLang.ts`**：純函數語言偵測，**回傳 locale code**(便於單元測試與
  擴充)。第一版用「Unicode script 區段比例」規則(CJK→`zh`、Hiragana/Katakana→`ja`、Hangul→`ko`、
  其餘 Latin→`en`)。新增語系時只在此檔加規則,不動其他程式。內容無法判定時回 `und`(unknown)。

## 多語系擴充設計(關鍵 —— 為「未來新增語系」預留)

整條語言鏈一律以 **locale code 字串**為單位,不寫死 ZH/EN 二分:

- **`detectLang(md): string`** —— 回傳 `zh`/`en`/`ja`/`ko`/…/`und`。靠 Unicode script 區段比例判定,
  加語系=加一條規則。比 ZH/EN 二分更可擴展。
- **BFF 端點 `?lang=` 接受任意 locale code**;不給則回全部。回傳的每張問卷帶 `language` 欄位。
- **前端 `navigator.language` → locale**:`navigator.language`(如 `ja-JP`)取 primary subtag(`ja`)
  當訪客偏好語言;廣場預設以此篩選,並提供「全部語言」與(未來)語言下拉。
- **介面語言 vs 內容語言解耦**:目前 UI 僅 ZH/EN([LanguageContext](frontend/src/context/LanguageContext.tsx)),
  但**問卷內容語言可超出 UI 支援範圍**。廣場篩選用「內容 locale」,不受 UI 只有兩語的限制 ——
  未來 UI 加語系時,廣場無需改動即可篩出新語系問卷。
- **建立問卷的語言**:第一版由 `detectLang` 自動判定(對上使用者選的「自動偵測」)。未來若要讓創建者
  手動覆寫,只需在 frontmatter 加選填 `language: <locale>` 欄位,BFF 優先用它、否則 fallback 到自動偵測 ——
  此擴充點已預留,本次不實作。

## 驗證方式(end-to-end)

1. **合約不變**：確認 `contracts/` 無改動,沿用現有 packageId。
2. **發布公開問卷**：CreatePage 開「公開到廣場」→ FundPage 發布。用 Sui explorer / `getObject`
   確認 `encrypted_content` 第一個 byte = `0x00` 且其後為可讀 markdown。
3. **BFF 端點**：`curl 'http://localhost:3100/api/surveys/explore?lang=zh'` 應回傳該問卷,
   `language` 欄位正確;發一張英文問卷確認被歸類 `en`,`?lang=zh` 時不出現、`?lang=en` 時出現。
4. **私密問卷不外洩**:發一張未勾選公開的問卷,確認**不**出現在廣場回應中。
5. **訪客流程**:登出/未連錢包,瀏覽器語言設為 zh,進 `/explore` → 預設只看到中文問卷;
   切「全部語言」看到全部;點卡片進 `/s/:id` 能直接看到題目(無 URL 金鑰也能讀)。
6. **填答門檻不變**:對 `min_tier>0` 的公開問卷,訪客可瀏覽但提交時仍要求對應 SurveyPass。
7. **單元測試**:`detectLang` 對中/英/混語樣本;BFF handler 對 mock 的公開/私密/封存問卷做篩選。
8. **回歸**:既有私密問卷的加密/解密與 `/s/:id` 填答流程不受影響(`0x01` 路徑)。

## 風險與備註

- **語言偵測準確度**:用 Unicode script 區段比例(回 locale code)即可,無需第三方庫;短文字/混語
  邊界情形可接受。未來要更準或支援更多語系,只需擴充 `detectLang` 規則或換成輕量偵測庫,介面不變。
- **RPC 成本**:廣場需 `multiGetObjects` 讀問卷+金庫物件,靠 60s 目錄快取攤平,當前測試規模無虞;
  問卷量大時再考慮輕量索引層(目前 BFF 無 DB,不在本次範圍)。
- **舊測試問卷**:無格式標記者一律當加密處理,不會誤上架;必要時手動重發為公開問卷。
- **公開=永久公開**:明文內容上鏈不可逆,CreatePage 開關需明確警示創建者。
