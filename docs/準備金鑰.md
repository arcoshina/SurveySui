# SurveySui 金鑰產生操作手冊

> 本文件說明如何**產生**上線所需的各類金鑰,銜接 [上線部署指南.md](./上線部署指南.md) §1.2 的金鑰分層表。
> 分層、存放與輪換規範以 [安全指引.md](./安全指引.md) 為準;`.env` 變數總覽見 [安全指引.md §3](./安全指引.md) 與 repo 根目錄 [.env.example](../.env.example)。
>
> ⚠️ **所有產生指令請在你自己的終端執行。** 指令輸出**即私鑰明文**,切勿貼到公開頻道、Issue、聊天或截圖;切勿 commit 進版控。

---

## 0. 總覽

| 金鑰變數 | Tier | 產生方式 | 去向 |
| :--- | :--- | :--- | :--- |
| `SUI_ADMIN_PRIVATE_KEY` | 0 | Sui CLI(`sui keytool`) | 僅本機部署終端 |
| `SURVEY_PASS_ISSUER_PRIV` | 1 | `openssl rand -hex 32` | BFF(+ `do` 模式 Worker) |
| `GAS_SPONSOR_PRIV_1` / `_2` | 1 | `setup-multisig-sponsor.ts`(一次產生) | BFF + Gas Worker |
| `GAS_SPONSOR_PUBKEY_3` | 1 | 同上(公鑰,非機密) | BFF + Gas Worker |
| `GAS_SPONSOR_PRIV_3_OFFLINE` | 1 | 同上 | **僅線下冷儲存** |

另需 Tier 2 高熵秘密(`ADMIN_SECRET`、`SURVEY_PASS_ISSUER_SALT`),見 §4。

**建議產生順序**:Admin → Issuer → Multisig → Tier 2 秘密。

## 1. `SUI_ADMIN_PRIVATE_KEY`(Tier 0)

部署 Move 合約、初始化 AMM 資金池、合約升級、緊急 rescue 用。

```bash
# 產生一個新的 ed25519 地址
sui client new-address ed25519

# 匯出該地址的私鑰(取得 suiprivkey1... 字串)
sui keytool export --key-identity <上一步產生的地址>
```

- 匯出值為 `suiprivkey1...` 格式;程式亦接受 hex(見 [scripts/src/init.ts:479](../scripts/src/init.ts#L479) 依 `suiprivkey` 前綴判斷)。
- 將私鑰填入**本機部署用** `.env` 的 `SUI_ADMIN_PRIVATE_KEY`,對應地址填 `SUI_ADMIN_ADDRESS`。
- 部署前先讓此地址持有足夠 SUI 支付部署 gas。

🚫 **絕不可**放入 BFF 運行環境 — BFF 啟動會 `delete` 此變數,且 `assertSecureEnv()` 偵測到即直接 crash(見 [安全指引.md §6.1](./安全指引.md))。

---

## 2. `SURVEY_PASS_ISSUER_PRIV`(Tier 1,簽 Pass ticket)

簽發 SurveyPass 准考證 Ticket(Ed25519)。程式讀取 **32-byte hex seed**(見 [bff/src/auth/ticket.ts:70-77](../bff/src/auth/ticket.ts#L70-L77),hex 解碼後取前 32 bytes)。

```bash
openssl rand -hex 32
```

- 輸出 64 字元 hex,填入 `SURVEY_PASS_ISSUER_PRIV`。
- 放 BFF 運行環境;若 `GAS_STATION_MODE=do`,Gas Worker 也需要(僅用於重驗 ticket)。
- 對應公鑰需綁到鏈上 `ISSUER_CONFIG`(由部署/重置流程的 `set_issuer_pubkey` 寫入,見 [scripts/src/reset-registry.ts](../scripts/src/reset-registry.ts))。

🚫 **只**簽 ticket,**不可**兼作 gas 代付帳戶。

## 3. Gas 代付 2-of-3 Multisig(Tier 1,四個變數)

`GAS_SPONSOR_PRIV_1`、`_2`、`GAS_SPONSOR_PUBKEY_3`、`GAS_SPONSOR_PRIV_3_OFFLINE` 必須**一次產生**,因為它們要組成同一個 multisig 地址。請勿各自單獨產生。

```bash
pnpm --filter @surveysui/scripts exec tsx src/setup-multisig-sponsor.ts
```

腳本(見 [scripts/src/setup-multisig-sponsor.ts](../scripts/src/setup-multisig-sponsor.ts))輸出分三區塊:

### (a) Runtime 機密 → BFF `.env` + Gas Worker
```
GAS_SPONSOR_PRIV_1=...            # K1 熱簽名私鑰
GAS_SPONSOR_PRIV_2=...            # K2 熱簽名私鑰
GAS_SPONSOR_PUBKEY_3=...          # K3 公鑰(非私鑰,可上線)
GAS_SPONSOR_MULTISIG_THRESHOLD=2
GAS_SPONSOR_ADDRESS=0x...         # 代付池地址,充值就轉到這裡
```

### (b) 離線冷備 → 嚴禁上雲
```
GAS_SPONSOR_PRIV_3_OFFLINE=...    # K3 私鑰:僅密碼管理器 / 紙本 / 冷錢包
```

### (c) Cloudflare Worker 注入(僅 `do` 模式)
```bash
cd workers/gas-station
wrangler secret put GAS_SPONSOR_PRIV_1
wrangler secret put GAS_SPONSOR_PRIV_2
wrangler secret put GAS_SPONSOR_PUBKEY_3
```

安全重點:
- K3 **私鑰**(`_OFFLINE`)離線保存後不要出現在任何 `.env` / 雲端;只有 K3 **公鑰**(`PUBKEY_3`)上線。
- threshold=2:單一熱鍵(K1 或 K2)外洩無法單獨動用資金池。
- 充值請轉到輸出的 `GAS_SPONSOR_ADDRESS`(multisig 地址),**不是**舊單簽地址。

## 4. Tier 2 高熵秘密

| 變數 | 產生 | 備註 |
| :--- | :--- | :--- |
| `ADMIN_SECRET` | `openssl rand -hex 32` | 管理 API Bearer;隨時可輪換 |
| `SURVEY_PASS_ISSUER_SALT` | `openssl rand -hex 32` | Nullifier pepper;**有用戶後極難輪換**(見 [安全指引.md §7.3](./安全指引.md)),須 ≥12 字元且非 `default_salt` |

OAuth / World ID / Resend 等憑證請於各自 SaaS 後台取得(見 [安全指引.md §3.3](./安全指引.md))。

---

## 5. 安全紅線

- 私鑰/secret **不得** commit、不貼公開頻道、不入 CI 明文(用 GitHub Encrypted Secrets)。
- `GAS_SPONSOR_PRIV_3_OFFLINE` **嚴禁**部署至任何雲端或伺服器。
- BFF 環境**不得**含 `SUI_ADMIN_PRIVATE_KEY`。
- dev / staging / prod 使用**不同**金鑰與 salt,禁止共用。
- 完成後對照 [安全指引.md §8 上線前檢查清單](./安全指引.md) 逐項核對。
