## 空白問卷解密金鑰存取策略（S4.2 延後評估）

目前問卷連結格式為 `/s/<vaultId>#<contentKey>`，`contentKey` 存於 URL fragment，是受訪者解密題目內容的唯一入口。

**已確認的設計決策**：
- 發起人若遺失連結，可透過在 `encrypted_content` blob 末尾附加 sealed key（以發起人 X25519 公鑰加密的 `contentKey`）從 vault ID 還原，不需合約異動（Option A）。
- 受訪者仍需從連結取得 `contentKey`；若要改為「持有有效 SurveyPass 即可換取」，需引入以下其中一種機制，評估留至下一版：
  - **BFF 金鑰伺服器**：鏈上驗證 Pass 後由 BFF 回傳 `contentKey`（中心化，實作較簡單）
  - **Seal（Mysten 閾值加密服務）**：policy 寫在 Move 合約，去中心化，與 SurveyPass 整合後 URL 可完全不攜帶金鑰（需等 S5.1 SurveyPass 設計定案）
