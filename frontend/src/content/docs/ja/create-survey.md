# アンケートの作成

全体の流れ：アンケートの設計 -> 報酬の設定 -> 資金注入してオンチェーン公開 -> 進捗の管理 -> クローズと締め。

## ステップ1：アンケートの設計

アンケートは Markdown エディタで作成し、4種類の設問タイプに対応します：

- 単一選択：回答者が1つを選ぶ。
- 複数選択：複数選択が可能。
- テキスト：自由入力。
- スケール：例として1〜5点の満足度。

選択式の設問は最大50個の選択肢に対応します。
選択肢のランダム並べ替えにも対応し、並び順によるバイアスを低減します。

アンケートの設問を *ローカルで暗号化* してからオンチェーンにすることを選べます。10 KB 以内の設問はそのままオンチェーンに保存され、より大きなアンケートは自動的に Walrus へ保存されるため、大きなアンケート用に保存費用として Walrus トークンを用意する必要があります。いずれの方式でも、アンケートのハッシュ値は金庫にバインドされます。**アンケートは一度公開すると変更できません。**

## ステップ2：参加条件と報酬の設定

締切：アンケートの有効期間は最長3か月で、期限が来ると自動的に受付を停止します。
受け入れる検証ソース：どの検証ソース（Email、Google、GitHub、World ID）を持つユーザーを受け入れるか選べます。または特定の NFT の保有を要求できます。要求する認証が強い（World Orb など）ほど偽アカウントでの水増しは難しくなりますが、対象者も少なくなります。

報酬ルール：
- 1回の回答あたりの報酬額（初回回答者が受け取る額）。
- 受け入れる最大人数（予算上限 = 人数 × 1回の報酬）。
- 重複回答（任意）：同一人物が再度回答し、より低い重複報酬を受け取ることを許可します。追跡型の調査に適します。

## ステップ3：資金注入してオンチェーン公開

設定が完了すると資金注入ページに進み、システムが推定総コストを提示します。
確認後、以下の動作が同一のインタラクション内で完了します（すべて成功するか、まとめてキャンセルされ、「一部のステップだけ失敗」する中間状態は生じません）：

1. SUI を投入して報酬トークン SSR を鋳造する。
2. アンケート専用の金庫（Vault）を作成し、SSR の報酬を注入する。
3. gas 補償リザーブを注入する。
4. プロトコル手数料 20% を支払う（現在は半額）。
5. アンケート内容を登録する（ハッシュを金庫にバインド）。
6. 金庫を一般の回答に向けて開放する。

完了するとアンケートのリンクが得られ、そのまま共有できます。

ヒント：アンケートの設問も暗号化している場合、URL の `#` 以降の部分が復号鍵です。むやみに外部に漏らさないようご注意ください。

## ステップ5：クローズと締め

ダッシュボードはリアルタイムで表示します：回答済み人数、完了率、支払い済み報酬、残り予算、締切までのカウントダウン。

クローズ（Close）：主催者はいつでもクローズできます。クローズ後、残りの報酬予算と gas リザーブはあなたのウォレットへ返却され、アンケートは受付を停止します。締切を過ぎると、誰でもクローズを起動できますが、資金は同じくあなたへ返却されます。

結果の閲覧：結果ページであなたの鍵を使ってすべての回答を復号でき、基本的な統計が組み込まれています。

バーン：クローズ後、アンケートの設問と**回答**をバーンできます。バーンは回答500件ごとのバッチで分割実行され、最後のバッチで金庫とアンケートオブジェクトをバーンし、オンチェーン保存に占有していた押金（storage rebate）を回収します。放置し続けると、プラットフォームが猶予期間の経過後に自動で整理します。実行コストを差し引いた余剰は *50% を手数料として徴収* されますので、必ずデータのバーンを忘れないようにしてください。
