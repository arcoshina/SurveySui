# Overflow 2026 ハイライト

## 回答即受取のプログラマブル決済

SurveySui は シビル対策の検証 -> 回答のオンチェーン記録 -> 報酬の支払い を1回の PTB インタラクションにまとめ、報酬付きアンケートを回答者・主催者の双方にとってシンプルかつ効率的にします。

SurveySui では、回答した後に主催者が送金してくれるのを待つのではなく、資格を自動的に検証し、水増しを防ぎ、条件が成立した瞬間に支払う条件付き決済が行われます。報酬の支払いは契約により条件付きで実行され、主催者やプラットフォームを信頼する必要はありません。

## What We Built

小規模事業者向けの**オンチェーンアンケートプラットフォーム**です。主催者は Markdown でアンケートを書き、報酬予算を注入します。暗号資産に不慣れな回答者でも、SUI を持たないまま初回の回答・受取フローを完了できます。アンケートにおける「回答の収集」と「報酬の支払い」は、煩雑で高コストな一連の手順ではなく、契約が完了を保証する1回の情報トランザクションになります。

| 役割 | ニーズ | やること |
|------|--------|-----|
| 主催者 | 市場情報の収集 | 資金注入、宣伝、情報整理 |
| 回答者 | 意見の表明 | 回答、SSR の受取 |
| 管理者 | サービスの運営 | システムと報酬リザーブプールの維持 |

## Why We Need It

### 効率が低く浸透率もまだ低い市場

「報酬を提供すると、アンケートの回収率が顕著に向上し、情報もより詳細になる」。これは信頼できるサンプルに対価を払う意思のある、明確なニッチ市場です。

アンケートは投稿分析よりも細やかな情報を提供できます。しかし単一のアンケートのデータには大きな価値はなく、アンケートシステムには高い摩擦があります。送金不要のギフトカードを選んだとしても、回答者は受取の閾値に達するまで多くのポイントを貯める必要があります。多くの回答者は閾値に届かずに諦め、一部の人は労力と見返りが釣り合わないと感じ、詐欺に遭っているのではと疑うことさえあります。

SurveySui はブロックチェーンの効率で既存のプロセスを作り直し、回答者が回答後すぐに報酬を受け取れるようにします。回答者は閾値まで貯めなくても引き出せ、高額な手数料も負担しません。

報酬付きアンケートの主流の収益モデルは「完了サンプル1件ごと」の従量課金で、単価には回答者の報酬コストとプラットフォームの取り分が含まれます。報酬そのものがこのビジネスの中核エンジンであり、そのエンジンの信頼性こそ、オンチェーンが価値を付加できる部分です。

## How It Works

### トークンエコノミー

プラットフォームトークンは **SR**（Survey Reward）です。主催者は SUI を費やしてトークンプールから SR を鋳造します。鋳造された SR は自動的にプールへ**ロック**され、主催者はその権利を表す **SSR**（Staked Survey Reward）を取得します。主催者は SSR を回答者への報酬として、アンケート専用の金庫に事前注入します。回答者が取得した SSR は自由に流通できますが、トークンプールから直接 SUI に戻すことはできません。

SR の鋳造時に投入された SUI も、リザーブとして同時にトークンプールにロックされます。SR の鋳造価格はプール内の SR / SUI 比率によって決まります。プロジェクト側はプール内の SUI をアンロックし、市場で SSR を買い戻し、プール内の SR と SSR を同量バーンすることで、市場を安定的かつ予測可能に保てます。第三者は任意に交換プールを作成でき、プロジェクト側が干渉する手段はありません。

### SurveyPass シビル対策設計

各ウォレットは soulbound な **SurveyPass** を1冊保持し、複数の身元ソース（Email、OAuth、World ID）を同時にバインドできます。SurveyPass は検証済みの身元をソルト付きの一方向ハッシュにして記録に書き込み、「1身元1ウォレット」をグローバルにバインドします。各アンケートではさらに独立した nullifier を別途ハッシュし、重複回答を防ぎつつ、アンケート横断での関連付けを困難にします。

ウォレットを変更する必要がある場合、SurveyPass 全体をバーンし、新しいウォレットで検証を再鋳造できます。SurveyPass のバーンにプロジェクト側の許可は不要で、所有者はいつでも有料で削除できます。

### アンケートの暗号化・復号とセキュリティ設計

暗号化型アンケートの本文は `AES-256-GCM + URL フラグメント鍵` で暗号化され、チェーンには暗号文・設問ハッシュ・設問タイプ構造を保存します。回答は `X25519 + ポスト量子 ML-KEM-768` のハイブリッド暗号化を用い、主催者はウォレットだけで復号できます。

フロントエンドは Markdown のレンダリング時に、すべての生 HTML をエスケープし、javascript: / data: などの悪意あるリンクプロトコルを遮断し、外部画像は一律バックエンドのプロキシ経由とします——これにより XSS、スクリプト、リンクインジェクションを排除します。コンテンツのハッシュはオンチェーン後に改竄不可となり、完全性検証を提供します。ハッシュ値の不一致を検出した場合は警告ページを表示しますが、回答を強制的にブロックはしません。

### Gas 代払いの仕組み

回答に必要な gas はまずバックエンドが代払いし、同時に主催者が事前にプールした vault からバックエンドへ補償します。vault が尽きるかバックエンドがオフラインになっても、回答者は自分で gas を払って回答することを選べます。

## Sui / Move 機能の活用

### 1. PTB によるアトミックな資金フロー

多段階の資金フローを1回のトランザクションに圧縮してアトミックに完了させ、中間の不整合状態が生じません。金庫は資金注入が完了するまで単一所有者オブジェクトのまま維持され、注入完了後に shared object として公開され到達可能になります。その間は外部から到達できません。claim の資格認可は、バックエンドのルールではなく Pass の型と nullifier によってオンチェーンで強制されます。Sponsored Transactions と組み合わせることで、回答者は全行程で残高0、署名は1回だけで済みます。

### 2. Soulbound Pass ＋ nullifier によるシビル対策

資格検証はすべて契約上にあり、バックエンドは身元証明を発行するだけなので、バックエンドが侵害されても資格を偽造したり報酬を二重に受け取ったりはできません。ソルト付きの一方向ハッシュ nullifier で、グローバルな「1身元1ウォレット」バインドと各アンケートの重複排除を行います。チェーンにはハッシュのみを保存し個人情報は保存せず、オンチェーンデータから本人を逆算することはできません。Pass の譲渡不可は、バックエンドのルールではなく型システム（`store` なしの `has key`）によって保証されます。

### 3. オンチェーンリソースの活用 — storage rebate

データの強制削除：フロントエンド UI + 高い延滞手数料により、主催者にオンチェーン領域の解放を促します。アンケート終了後3か月経っても主催者がデータをバーンしない場合、バックエンドが強制的にバーンし、コストを差し引いた後の押金の50%を手数料として徴収します。バックエンドが機能しない場合、1年後には誰でもバーンを起動できるようになります。大きな設問は Walrus へ振り分け、チェーンには blobId インデックスのみを残します。

## 今後の展望

- ロック資産の活性化：SUI に加え、SuiUSDe のような利回りを生む資産をトークンプールに追加し、ロックされたリザーブを静的な担保だけにとどめません。これらの資産が継続的に生む収益が SR の価値を支え、追加発行なしでトークンプールをより堅牢にします。
- より細やかな顧客層分類：主催者は回答者が自己申告する年齢・地域・職業などの属性に基づき、アンケートの回答条件を設定できます。これにより報酬予算を狙った顧客層へ正確に投下し、回収サンプルの代表性と商業的価値を高めます。
- シビル対策の強化：既存の多重身元バインドに加え、KYC 級の重複しない実在人物検証を導入し、大規模なシビル攻撃をさらに防ぎます。高単価・高機微なアンケートでは、各サンプルが唯一かつ実在の回答者から来ることを保証できます。
- コミュニティ機能の追加：プラットフォームは回答者の回答履歴や興味タグに基づき、適切なアンケートを能動的にマッチングし推薦します。これにより主催者と対象回答者の距離を縮め、回答者の参加率と長期的な定着を高めます。
