# Overflow 2026 亮點

## 即答即領的程式化支付

SurveySui 把 防女巫驗證-> 答卷上鏈-> 獎勵發放 打包在一筆 PTBs 交互內完成，也讓有獎徵答這件事，對於問答雙方都變得簡單高效。

在 SurveySui，不再只是填答之後等待問卷業者把錢匯出，而是一筆會自動驗證資格、防止灌水、並在條件成立時立刻放款的條件式支付。獎勵發放由合約條件式執行、無需信任發起者或平台。


## What We Built

一個給小型創業者的**鏈上問卷平台**：發起者用 Markdown 寫問卷、注入獎勵預算；即便是非幣圈玩家的受訪者，也能在沒有 SUI 的情況完成初步的填答領獎。問卷的「收答案」與「發獎勵」不再是一串繁瑣、高成本的步驟，而是合約保證完成的一筆資訊交易。

| 角色   | 需求         | 做的事               |
| ------ | ------------ | -------------------- |
| 發起者 | 蒐集市場資訊 | 注資、宣傳、整理資訊 |
| 受訪者 | 表達意見     | 填答、領 SSR         |
| 管理者 | 營運服務     | 維護系統和獎勵儲備池 |

## Why We Need It

### 缺乏效率且滲透率仍低的市場

「提供獎勵時，問卷的回收率顯著提升、資訊也更詳細」。這是一個明確的利基市場，願意為可靠的樣本付費。

問卷可以提供比貼文分析更細緻的資訊。然而，單一問卷的資料沒有太多價值，問卷系統也有較高的摩擦。即便選擇領取不用匯款的禮品卡，受訪者也需要積累許多點數才能達到領獎門檻。許多受訪者因為達不到門檻而放棄繼續參與，甚至一部分人認為投入與付出不成比例，懷疑自己是不是遇到詐騙。

 SurveySui 希望用區塊鏈的效率重造既有流程，讓受訪者填完馬上領獎。受訪者不用等積累到門檻才能提領，也不用負擔高額手續費。

而獎勵問卷的主流收入模式是「每完成一份樣本」按件計價，單價內含受訪者獎勵成本與平台抽成。獎勵本身就是這門生意的核心引擎——而引擎的可靠度，正是鏈上能加值的地方。


## How It Works

### 代幣經濟
  
平台代幣是 **SR** (Survey Reward)。發起者花費 SUI 向代幣池鑄造 SR，SR 鑄出即自動**鎖定**進池，發起者則取得表彰這份權利的 **SSR** (Staked Survey Reward)。發起者把 SSR 作為受訪者的獎勵，預先注入問卷的獨立金庫。受訪者 取得的 SSR 則可以自由流通，但不能直接跟代幣池換回 SUI。

SR 被鑄造時投入的 SUI 也同時鎖在代幣池中作為儲備。SR 的鑄造價格由代幣池內的 SR / SUI 比率決定。項目方可解鎖池內的 SUI，再到市場上買回 SSR，並等量銷毀代幣池中的 SR 和 SSR，藉此維持市場穩定可預期。第三方可以任意建立兌換池，項目方無能力干涉。


### SurveyPass 防女巫設計

每個錢包持有一本 soulbound 的 **SurveyPass**，可同時綁定多種身分來源（Email、OAuth、World ID）。SurveyPass 將驗證的身分加鹽作單向雜湊後寫入驗證，全域綁定「一身分一錢包」；每份問卷各別再雜湊一組獨立的 nullifier，在確保不重複填的同時、也不易跨問卷關聯。
使用者需要更換錢包時，可銷毀整本 SurveyPass 再至新錢包重新鑄造驗證。銷毀 SurveyPass 不需要項目方授權，所有者在任何時候都可以付費刪除。


### 問卷的加解密及安全設計

加密型題目卷的內文使用 `AES-256-GCM + URL fragment 金鑰` 加密，鏈上存密文、題目雜湊與題型結構；答卷用 `X25519 + 後量子 ML-KEM-768` 混合加密，發起者用錢包就能解密。

前端渲染 Markdown 時會跳脫所有原始 HTML、攔截 javascript: / data: 等惡意連結協定、外部圖片一律走後端代理——杜絕 XSS、腳本與連結注入。內容雜湊上鏈後不可竄改，提供完整性驗證。偵測到雜湊值不符時，將跳出警告頁但不硬性阻斷填答。


### Gas 代付機制

受訪者填答需要的 gas 先由後端代付，同時由發起者預存的 vault 補償後端代付。若 vault 用盡或後端離線，仍可以由受訪者決定自付 gas 填答。


## Sui / Move 特性運用

### 1. PTB 原子金流

把多步金流壓成單一筆交易內原子完成，沒有中間不一致狀態。金庫在配資完成前維持為單一擁有者物件、配資完成才公開為 shared object，期間外界不可達；claim 的資格授權由 Pass 型別與 nullifier 在鏈上強制，而非後端規則。搭配 Sponsored Transactions，受訪者全程餘額為 0、只需簽一次名。


### 2. Soulbound Pass ＋ nullifier 防女巫

資格驗證全部落在合約，後端只簽發身分憑證，即使後端被攻破也無法偽造資格或重複領獎。以加鹽單向雜湊的 nullifier 做全域「一身分一錢包」綁定與每份問卷的去重，鏈上只存雜湊、不存個資，無法從鏈上資料逆推真實身分。Pass 的不可轉讓由型別系統保證（`has key` 無 `store`），而非後端規則。


### 3. 鏈上資源利用 — Storage rebate

強制刪除資料：前端 UI + 高逾期手續費，引導發起人釋放鏈上空間。若發起人在問卷結束後三個月仍未銷毀資料，後端將強制銷毀並收取 50% 扣除成本後的押金作為手續費。若後端失效，一年後開放為任何人皆可觸發銷毀。大型題目分流至 Walrus、鏈上只留 blobId 索引。


## 未來展望

- 活化鎖倉資產：除了 SUI，代幣池中也加入類似 SuiUSDe 的生息資產，讓鎖倉的儲備不再只是靜態擔保。這些資產持續產生的收益可回流支撐 SR 的價值，使代幣池在不額外增發的前提下更加穩健。  
- 更細緻的客群分類：發起者可依年齡、地區、職業等受訪者自申報的屬性，設定問卷的填答門檻。如此能把獎勵預算精準投放到目標客群，提升回收樣本的代表性與商業價值。  
- 強化反女巫機制：在現有多重身分綁定之上，導入 KYC 等級的不重複真人驗證，進一步杜絕大規模女巫攻擊。針對高單價或高敏感度的問卷，可確保每份樣本都來自唯一且真實的受訪者。  
- 加入社群機制：平台依受訪者的填答歷史與興趣標籤，主動媒合並推薦合適的問卷。這能縮短發起者與目標受訪者之間的距離，同時提升受訪者的參與率與長期留存。
